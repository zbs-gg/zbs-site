01The real problem

It's not that it forgets. It's that the right memory doesn't fire when it should.

The same question needs a different memory depending on the moment — what you're doing, where you are in the work, who's asking. Storage is solved. Retrieval-for-this-state is not.

same query · different state · different right answer
"how do we handle auth here?"
state — mid-incident, prod down
surface the rollback runbook + the decision "we dropped sessions for JWT, here's why".
"how do we handle auth here?"
state — planning a new service
surface the auth ADR + the do-not-repeat note from the last migration.
A flat vector store hands back the same nearest-text both times. State-aware retrieval picks the memory this moment actually needs — that's the axis the lab builds and measures.

You've read the trillion "build your second brain" threads. They organize one person's notes. They don't make the right thing fire mid-work, and they say nothing about a team. That's not where the difficulty is — and it's not what this lab is.
