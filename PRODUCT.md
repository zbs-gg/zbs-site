# Product

## Register

brand

## Users

Technical friends, AI-heavy builders, partner reviewers, and investor-adjacent
people evaluating a Claude Code-first Pulse preview. They are willing to run a
developer preview, but they should not have to decode memory infrastructure
before seeing the value.

## Product Purpose

The site explains ZBS/Garden surfaces and, for `/preview`, gives Pulse MCP a
clean install-by-link entry point. Success means a technical user understands:
Pulse is local, backend model calls are off by default, raw transcript capture
is off, one memory can be saved, the next resume can be inspected, and memory can
be wiped.

## Brand Personality

Quiet, alive, careful.

## Anti-references

Do not look like a generic MCP server setup, SaaS admin dashboard, crypto graph,
developer console, AI cube/neural-net landing page, or import-first migration
tool. Avoid cold blue SaaS as the primary feeling.

## Design Principles

1. First contact should feel like a small local memory beginning to breathe.
2. Trust comes before spectacle: show what will be injected and how to wipe it.
3. Proof beats explanation: one memory, one fresh session, one viewer.
4. Import and graph complexity stay secondary until the first proof lands.
5. Warmth is allowed only when it stays honest and user-controlled.

## Accessibility & Inclusion

Target WCAG AA contrast, readable mobile command blocks, reduced-motion support,
and no claims of emotional certainty. Emotion/state language must remain
optional, editable, and private by default.
